# Carnival Dash 2.0 integration guide

Follow these steps after creating the **Game** template project in Xcode (UIKit lifecycle).

## 1. Remove the template SpriteKit scene
Delete the autogenerated `GameScene.swift` file that ships with the template. All gameplay scenes will come from the files listed below.

## 2. Add the provided files to your target
Drag the Swift files from this folder into the matching groups shown in Xcode (select **Copy items if needed** and ensure the `Carnival-Dash 2.0` target is checked).

| Xcode Group | Files to place inside |
|-------------|----------------------|
| **AppDelegate** | `AppDelegate/AppDelegate.swift` |
| **GameViewController** | `GameViewController/GameViewController.swift` |
| **Main** (or create a new **Shared** group) | `Shared/Shared.swift` |
| **GameScene** (rename the group to **Scenes** for clarity) | `Scenes/MainMenuScene.swift`<br>`Scenes/ParkScene.swift`<br>`Scenes/BottleTossScene.swift`<br>`Scenes/PlinkoScene.swift` |

> Tip: After renaming the GameScene group to **Scenes**, make sure all four scene files are members of the `Carnival-Dash 2.0` target (File Inspector ▶ Target Membership).

## 3. Build settings tweaks
* In **Deployment Info**, keep only **Portrait** checked for iPhone orientations.
* No other Info.plist changes are required.

## 4. Run the project
Build and run. The flow should be:
1. Launch → **MainMenuScene** shows title + Play button.
2. Tap **Play** → **ParkScene** hub with player movement, D-pad, keyboard WASD (Simulator), HUD tickets.
3. Step onto yellow/purple pads and tap to open Bottle Toss / Plinko mini-games. Use Back buttons to return.

Tickets persist between scenes via `UserDefaults` and are displayed in each HUD.

---
_All placeholder art uses simple colored SpriteKit shapes so the project runs without external assets._
